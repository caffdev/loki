# Feature Mockup Specs

This file is Pix's source of truth. Each feature listed here should have a corresponding `docs/html/<feature-name>.html` mockup.

## Format

Each entry follows this structure:

```
## <feature-name>
**Status**: pending | in-progress | done
**Primary user action**: <what the user is trying to do>
**Must show above the fold**: <key information>
**Type**: page | modal | panel | inline-component
**Notes**: <any constraints or context>
```
